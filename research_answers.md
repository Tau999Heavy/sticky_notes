Question 1: HTTP State Preservation, Authentication and Session Management

HTTP is a stateless protocol, which means that each request sent from a client to a server is treated independently. The server does not automatically remember information about previous requests from the same user. To maintain the state of an application across multiple request-response cycles, web applications use cookies. When a user visits a website, the server can generate a unique session identifier and store it in a cookie on the user's browser. The browser then sends this cookie back to the server with subsequent requests. 

The server uses the session identifier to retrieve information associated with that user, such preferences or items stored in a shopping cart. This approach allows the application to provide a continuous user experience despite HTTP being stateless. User authentication tends to rely on this mechanism. After someone successfully logs in, the server creates a session and associates it with the authenticated user. As long as the session remains valid, the user can access protected resources without repeatedly entering user name / password etc. Sessions typically expire or disconect after a period of inactivity or if the user logs out, improving security by reducing the risk of unauthorised access.

Question 2: Performing Django Database Migrations to MariaDB

MariaDB is a popular server-based relational database management system that can be used with Django's applications. To perform migrations to MariaDB, the database server must first be installed and configured. A database and a dedicated user account should then be created with appropriate permissions. Next, a suitable database connector, such as mysqlclient or MariaDB Connector/Python, must be installed in the Django environment. The Django project's settings.py file is then updated to use the MariaDB database engine and connection details, including the database name, username, password, host, and port. Once the configuration is complete, Django's migration system can be used to create and apply database schema changes. 

The python manage.py makemigrations command generates migration files based on changes made to models, while python manage.py migrate applies those changes to the MariaDB database. Django automatically creates or updates the required tables and relationships. After the migration process is complete, the database can be checked using MariaDB administration tools to confirm that the tables were created successfully. This process ensures that the database structure remains synchronised with the Django models throughout the application's development lifecycle.
